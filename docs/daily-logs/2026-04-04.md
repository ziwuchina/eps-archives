## VBS Execution Research

### Key Findings

**EPS VBS execution is NOT triggered via `ExecFunction("script xxx.vbs")`**
- `EPS_ExecFunction("script xxx.vbs")` returns `ret=0` (command received) but does NOT trigger `RunScript`
- `ExecFunction` is a command dispatch layer only; `script/...` subcommands are parsed but blocked if no valid script context
- This explains why all attempts to run VBS via `EPS_ExecFunction(cmd)` produce `ret=0` but no side effects

**SSERPTools.dll exports confirmed (base varies per session):**
- `SSERPTools!0x1B4A0` = `CSSERPToolsInterface::ExecFunction` — the command dispatcher
- `SSERPTools!0x8DA0` = `CSScriptHandleBase::RunScript` ( UINT RunScript(CString*, LPCSTR) ) — **actual script executor**
- `SSERPTools!0x8DB0` = `CSScriptHandleBase::RunScriptEx`
- `SSERPTools!0x8D80` = CSScriptHandleBase instance context area
- `SSERPTools!0x1B120` = `CSSERPToolsInterface::RegisterCommand` — BCG menu command registration

**SScriptCore.dll (factory pattern):**
- `SScriptCore!Instance()` returns the ScriptEngineFactory singleton
- `ScriptEngineFactory::RunScript(factory, path)` = `0x52c1b85` — the underlying VBS engine
- `ScriptEngineFactory::LoadScript(factory, path)` = `0x52c1ca5`
- Calling convention: thiscall (ECX=factory pointer), stack=path CString*

**How VBS scripts actually execute in EPS:**
1. User right-clicks project node → BCG popup menu
2. Menu item command sent via WM_COMMAND to SSERPTools window proc
3. `OnContextMenu` / command handler in SSERPTools receives command ID
4. Handler resolves script name via `BdcProjectMan_GetScriptMenuItem.vbs` (looks up from profileIni/Script-new folder)
5. `CSScriptHandleBase::RunScript(path, "OnClick")` is called with correct `this` pointer
6. VBS executes in EPS process with `SSProcess` / `SSParameter` COM objects available

**Script directory structure:**
- Primary: `D:\EPS2026G\DeskTop\顺义精灵 Script-new\` (42 menu items registered via `BdcProjectMan_GetScriptMenuItem.vbs`)
- Also: `D:\EPS2026G\DeskTop\顺义精灵 Script\` (82 user scripts, no menu registration)
- Script items: `.vbs` files with `Sub OnClick()` entry point
- EPS COM objects available in scripts: `SSProcess`, `SSParameter`

**Key insight:** The "this" pointer for `CSScriptHandleBase` must be valid. Calling `RunScript` as a free function (without correct object) causes access violations.
