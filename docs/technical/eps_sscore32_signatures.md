# EPS SSCore32.dll 函数签名恢复（port 10004）

## 执行结果概览

- IDA MCP `check_connection`：`{ok: true, count: 4}`
- 目标实例：`select_instance(port=10004)` 成功
- 已反编译指定函数：
  - `0x1007723F`（USBKEY/Token/Vercode URL 构造核心）
  - `0x10076F40`（上层调用）
  - `0x10075D8A`（HASP 登录/scope/RTC）
  - `0x100668AB`（WININET + urlmon 缓存/下载）
- 已执行字符串检索：`usbkeyid` / `token` / `vercode` / `HASP`
- 已执行 xrefs：`xrefs_to(0x1016447C)`（字符串 `%s?usbkeyid=%d&token=%s&vercode=%s`）

原始采集文件：`reports/eps_sscore32_step_raw.json`

## 关键证据（精简）

1) URL 构造字符串命中
- `0x1016447C`: `%s?usbkeyid=%d&token=%s&vercode=%s`
- `xrefs_to(0x1016447C)` -> `frm=0x1007731B`（位于 `sub_1007723F`）

2) HASP 相关字符串命中
- `hasp_windows_82156.dll`
- `NetHasp.ini`
- `<?xml version="1.0" encoding="UTF-8" ?><haspscope> ... ip="%s" ...`

3) 关键调用行为
- `sub_1007723F`：`CString::Format("%s?usbkeyid=%d&token=%s&vercode=%s", ...)`
- `sub_1007723F`：`sub_100371B1` 生成 token；`CAPIEncryptLongString` 生成 vercode
- `sub_10075D8A`：`hasp_login` / `hasp_login_scope` / `hasp_get_rtc` / `hasp_hasptime_to_datetime` / `hasp_logout`
- `sub_100668AB`：`GetUrlCacheEntryInfoA` -> miss 时 `URLDownloadToCacheFileA`

## URL 构造完整调用链（核心链路）

- `sub_10076F40 (0x10076F40)` -> `sub_1007723F (0x1007723F)`
- `sub_1007723F` -> `sub_100371B1`（token）
- `sub_1007723F` -> `CAPIEncryptLongString`（vercode）
- `sub_1007723F` -> URL `"%s?usbkeyid=%d&token=%s&vercode=%s"`
