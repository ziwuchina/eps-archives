## 核心发现

### Themida保护阻止API调用
- `hasp_login()` 在外部调用时崩溃 (访问0x000140EC)
- `hasp_get_info()` 崩溃 (访问0x00000100)  
- `hasp_get_version()` 崩溃
- 只有 `hasp_get_size(handle, 0)` 返回9 (HASP_INVALID_HANDLE)

### HASP API返回值分析
- `hasp_login(int, int, handle)` → 访问违规 (Themida检测)
- `hasp_login(int, ulong, handle)` → 访问违规 (Themida检测)
- `hasp_login(string_vendor, handle)` → result=22 (参数错误)
- `hasp_get_size(any_handle)` → result=9 (INVALID_HANDLE)

### 内存搜索结果
- 种子数据不在.data段(未初始化数据区)
- 种子可能: (1)加密存储 (2)在堆/栈上动态分配 (3)每次运行时不同

## EPS 命令行自动化（2026-03-22 21:00-23:40）

### 核心发现
- EPS 命令行窗口: BCGPMiniFrame 类，标题含"命令行窗口"
- 命令输入框: Edit 控件（宽约1800px，高约18px）
- 命令输出框: 同一窗口内的另一个 Edit 或 Static 控件
- **正确清场**: 找到所有 #32770 对话框，按钮文本判断类型后处理

### 自动化流程（标准步骤）
1. **关所有旧弹窗**: 遍历所有 #32770，对"取消"按钮点取消，对"确定"点确定
2. **写命令**: WM_SETTEXT 直接覆盖输入框（不用 Ctrl+A，会产生多余 'a' 前缀）
3. **发回车**: PostMessage 异步发送 VK_RETURN
4. **等对话框**: 每0.5秒扫描一次 #32770，分析按钮判断类型
5. **识别对话框**:
   - 有"登录"+"取消"按钮 → 登录表单（用户名+密码Edit）
   - 只有"确定" → 结果/错误信息（读 Static 内容判断）
6. **填表单**: 点击 Edit 框，再逐字 WM_CHAR 输入（WM_SETTEXT 对 Edit 返回 -1）
7. **点按钮**: 根据按钮文本定位点击
8. **读结果**: 搜索 Static 文本含"已登录"为成功，含"失败/错误"为失败

### 关键 API
