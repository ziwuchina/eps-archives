## EPS EDB 数据读取 — 重大突破

### ODBC 成功连接 EDB
- **EDB文件 = 标准 Jet MDB 数据库**，可用 ODBC 直接读写
- 连接字符串：`DRIVER={Microsoft Access Driver (*.mdb, *.accdb)};DBQ=xxx.edb`
- Python库：`pypyodbc`（纯Python，无需原生驱动）
- ODBC driver 本身是 64位 Access 驱动，但 pypyodbc 通过它连接成功

### 表结构发现
| 表名 | 内容 |
|------|------|
| SSObject | 对象定义（ObjectID, FeatureID, LayerID, ObjectName） |
| SSObjectGeometry | 几何坐标（ObjectID, GeoCount, X, Y, Z） |
| SSObjectAttribute | 扩展属性（ObjectID, AttrName, AttrValue） |
| SSProcessLayer | 图层定义 |
| SSProcessSymbol | 符号定义 |

### 佛山数据.edb 内容（10个对象）
- ObjectID=1: 点，位于(独立)图层
- ObjectID=2: **使用权宗地**（唯一多边形），FeatureID=6803163，4个顶点
- ObjectID=6-13: 结构线（8条线对象），每个2个顶点

### 地籍号 vs 宗地号
- 宗地号 = 440605001011GB00019（有值）
- 地籍号 = NULL（数据库为空）
- DXF导出时EPS从宗地号截取前缀得到地籍号=440605001011
- 这是EPS运行时动态派生的，不是数据库原始值

### DXF vs EDB 数据对比
- DXF导出坐标与ODBC读取的EDB坐标**完全一致** ✅
- DXF属性（如地籍号）是EPS运行时填充的，与EDB原始数据可能不同

### EPS COM接口限制
- EPS的COM接口主要用于**创建对象**（CreateNewObjByCode等）
- **读取已有对象**的所有方法返回 err=438（方法不存在）
- `SetVisible(0)` 可隐藏窗口但不能真正无头运行（需要桌面会话）

```

    ↓ 调用

    ↓
```
